題目：3848. Check Digitorial Permutation

問題描述

digitorial 是指：一個數字的各個位數的「階乘總和(digitSum)」等於他自己 or 他自己的任何位數的排列組合（不能以0開頭）的數字，那他就被稱為 digitorial

例如 145, 541, 451 都是 digitorial ，因為 1! + 4! + 5! = 145，而這個 1! + 4! + 5! = 145 的結果可以個別從 145, 541, 451 身上得出，所以這三個數字都是 digitorial

題目要求：傳入一數字 n，回傳 n 是否是 digitorial

解題思路

其實可以反過來說，當找到 digitSum 的值本身就是 n 的一種排列組合時，正意味著 n
就是 digitorial
舉例而言想知道 451 是否為 digitorial，那我們只要計算：

  1. 451 的 digitSum -> digitSum 是 145
  2. 451 的每個位數分別是誰、有幾個 -> 分別由一個 1、一個 4、一個 5 所組成
    這時再看看 digitSum 分別由哪些位數組成？ -> 145 也是由「一個 1、一個 4、一個 5」所組成
    只要「 digitSum 的組成成分」 等於 「 n 的組成成分」就代表 digitSum 是 n 的其中一種排列組合了，那也就代表 n 的其中一種排列組合之 digitSum 會等於該排列組合的值 -> n 是 digitorial
  1. 先計算 n 的 digitSum 的值
  2. 檢查 n 和 digitSUm 的位數組成是否一致
  3. 如果一致，就代表 n 是 digitorial
    可以把計算階乘的邏輯另外寫成一個函數以簡化計算 digitSum 的程式碼

而檢查組成位數的函數 checkSame ，其內部的檢查邏輯適用一個長度 10 的陣列，存放 0~9 的出現次數（頻率）
先對 digitSum 取 10 的餘數紀錄每個數字的出現頻率（++）
再對 n 取 10 的餘數，計算每個數字的出現頻率，一旦出現就反向去 --，最後看陣列內若是有不為 0 的值，代表兩者組成成分不一致，回傳 false
